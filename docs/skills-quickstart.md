# 技能系统快速开始指南

## 概述

技能插槽系统为LLM智能体配备各种专业能力，让智能体可以通过Function Calling主动调用工具完成任务。

## 前置条件

### 1. 数据库迁移

在Supabase控制台的SQL Editor中执行以下SQL脚本：

```bash
docs/database-migration-skills.md
```

或者访问迁移API：
- **检查状态**: `GET /api/skills/migrate`
- **执行迁移**: `POST /api/skills/migrate`

### 2. 验证迁移成功

访问 `/api/skills/migrate` 确认表已创建：
```json
{
  "success": true,
  "data": {
    "agent_skills_exists": true,
    "skill_executions_exists": true,
    "migration_required": false
  }
}
```

## 快速开始

### 步骤1: 查看所有可用技能

访问 `/skills` 页面，查看所有预定义的技能：

**代码开发类**（5个技能）：
- 💻 代码生成 - 根据需求生成高质量代码
- 📄 文件创建 - 创建文件并写入内容
- 📁 目录创建 - 创建目录结构
- 📖 文件读取 - 读取文件内容
- ⚡ 命令执行 - 执行Shell命令

**文本处理类**（1个技能）：
- ✍️ 文案编写 - 生成各种类型的专业文案

**设计规划类**（1个技能）：
- 📋 PRD设计 - 生成完整的产品需求文档

**分析能力类**（1个技能）：
- 🔍 需求分析 - 分析和拆解用户需求

### 步骤2: 为智能体配置技能

**方法1: 通过智能体管理页面**
1. 访问 `/agents` 页面
2. 找到你想要配置的LLM智能体
3. 点击"配置技能"按钮
4. 切换技能开关
5. 点击"保存配置"

**方法2: 直接访问配置页面**
1. 访问 `/agents/[智能体ID]/skills`
2. 按类别浏览技能
3. 开启/关闭技能
4. 保存配置

### 步骤3: 在对话中使用技能

配置完成后，智能体在对话中会自动识别需要调用技能的场景：

**示例对话**：
```
用户: 帮我生成一个Python的Hello World程序

智能体: [检测到需要调用"代码生成"技能]
[执行代码生成技能]
[生成Python代码]

我可以帮你生成Python的Hello World程序：

```python
# Hello World - Python

def main():
    """打印Hello World消息"""
    print("Hello, World!")

if __name__ == "__main__":
    main()
```

这个程序定义了一个main函数，并在运行时打印"Hello, World!"消息。
```

### 步骤4: 查看技能统计

访问 `/skills/stats` 页面查看：
- 总体统计（执行次数、成功率、平均时间）
- 按技能统计（每个技能的详细数据）
- 执行趋势（每日使用趋势）
- 执行日志（最近的执行记录）

## 技能参数说明

### 代码生成
```json
{
  "language": "python",  // 编程语言
  "requirements": "创建一个Hello World程序",  // 需求描述
  "filename": "hello.py",  // 文件名
  "framework": ""  // 框架（可选）
}
```

### 文件创建
```json
{
  "file_path": "/path/to/file.txt",  // 文件路径
  "content": "文件内容",  // 文件内容
  "overwrite": false  // 是否覆盖已存在的文件

```

### 文案编写

## 常见问题

### Q1: 技能调用失败怎么办？

**可能原因**：
1. 智能体没有启用该技能
2. 参数不完整或类型错误
3. 技能执行权限不足（如文件写入权限）

**解决方法**：
1. 检查智能体技能配置页面
2. 查看技能执行日志 `/skills/stats`
3. 检查服务器日志 `/app/work/logs/bypass/app.log`

### Q2: 如何调试技能调用？

1. 查看技能执行日志，找到失败的记录
2. 检查params和result字段
3. 根据error信息定位问题

### Q3: 技能支持哪些编程语言？

代码生成技能支持：
- Python, Java, JavaScript, TypeScript
- Go, Rust, C++, C

### Q4: 如何添加自定义技能？

1. 在 `src/lib/skills/registry.ts` 中注册新技能
2. 实现技能的executor函数
3. 重启服务
4. 技能会自动出现在技能列表中

## 技能调用机制

### 自动识别
智能体会分析用户消息，自动判断是否需要调用技能。

### 参数提取
如果用户没有提供所有必要参数，智能体会：
1. 尝试从上下文推断
2. 询问用户缺失的参数

### 结果反馈
技能执行完成后：
1. 返回执行结果
2. 生成友好的总结
3. 提供后续操作建议

## 安全注意事项

1. **命令执行**技能具有系统访问权限，请谨慎使用
2. **文件操作**技能需要正确的路径权限
3. 建议在沙箱环境中测试技能调用
4. 定期审查技能执行日志

## 最佳实践

1. **为不同角色配置不同技能**
   - 开发者智能体：代码生成、文件操作
   - 产品经理智能体：PRD设计、需求分析
   - 运营智能体：文案编写

2. **定期查看技能统计**
   - 了解哪些技能使用最频繁
   - 识别性能瓶颈
   - 优化技能配置

3. **测试技能调用**
   - 在对话中测试各个技能
   - 验证参数传递正确性
   - 检查执行结果
